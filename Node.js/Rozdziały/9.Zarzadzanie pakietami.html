Zarządzanie Pakietami w Node.js

Node.js używa menedżerów pakietów, takich jak npm oraz yarn, które ułatwiają instalowanie bibliotek, zarządzanie zależnościami oraz konfigurowanie projektów za pomocą pliku package.json. W tej sekcji omówimy podstawowe operacje na pakietach, użycie npm i yarn, oraz tworzenie i zarządzanie plikiem package.json.

Tworzenie package.json

Plik package.json przechowuje informacje o projekcie oraz jego zależnościach. Aby stworzyć ten plik, można skorzystać z komendy npm init lub yarn init:


# Z użyciem npm
npm init

# Z użyciem yarn
yarn init

Komenda init poprowadzi przez konfigurację pliku package.json, pytając o takie informacje jak nazwa projektu, wersja, opis, punkt wejściowy (np. index.js), oraz lista autorów.

Instalowanie pakietów

Zarówno npm, jak i yarn umożliwiają instalowanie pakietów, które mogą być zależnościami projektowymi (zapisanymi w dependencies) lub zależnościami deweloperskimi (zapisanymi w devDependencies).

Instalowanie zależności


# Instalowanie zależności z npm
npm install [nazwa_pakietu]

# Instalowanie zależności z yarn
yarn add [nazwa_pakietu]

Instalowanie zależności deweloperskich

Zależności deweloperskie są używane tylko podczas procesu tworzenia aplikacji, np. narzędzia do testowania lub transpilacji kodu.


# Instalowanie zależności deweloperskich z npm
npm install [nazwa_pakietu] --save-dev

# Instalowanie zależności deweloperskich z yarn
yarn add [nazwa_pakietu] --dev

Zarządzanie zależnościami

Plik package.json przechowuje wszystkie zależności projektu, co ułatwia ich kontrolę i aktualizację.

Aktualizowanie pakietów

Aktualizacje pakietów pozwalają korzystać z najnowszych funkcji i poprawek bezpieczeństwa. Można je wykonać za pomocą komend:


# Aktualizowanie wszystkich pakietów z npm
npm update

# Aktualizowanie wszystkich pakietów z yarn
yarn upgrade

Usuwanie pakietów

Aby usunąć pakiet z projektu, można użyć odpowiednich komend:


# Usuwanie pakietu z npm
npm uninstall [nazwa_pakietu]

# Usuwanie pakietu z yarn
yarn remove [nazwa_pakietu]

Instalacja zależności z pliku package.json

Jeśli projekt już zawiera plik package.json, można szybko zainstalować wszystkie wymienione zależności:


# Instalacja zależności z npm
npm install

# Instalacja zależności z yarn
yarn install

Po wykonaniu powyższych komend wszystkie zależności zostaną zainstalowane na podstawie informacji zawartych w package.json.

Przykład pliku package.json

Poniżej znajduje się przykład pliku package.json zawierającego podstawowe zależności oraz zależności deweloperskie:


{
    "name": "moj-projekt",
    "version": "1.0.0",
    "description": "Przykładowy projekt Node.js",
    "main": "index.js",
    "scripts": {
        "start": "node index.js",
        "test": "echo \"Error: no test specified\" && exit 1"
    },
    "dependencies": {
        "express": "^4.17.1"
    },
    "devDependencies": {
        "nodemon": "^2.0.7"
    }
}

W tym przykładzie express jest zależnością produkcyjną, a nodemon - zależnością deweloperską, przydatną do automatycznego restartowania serwera podczas prac rozwojowych.

Podsumowanie

npm i yarn są podstawowymi narzędziami do zarządzania pakietami w Node.js. Dzięki nim można łatwo dodawać, aktualizować oraz usuwać zależności, a plik package.json pomaga w organizacji i automatyzacji procesu instalacji w projekcie.
