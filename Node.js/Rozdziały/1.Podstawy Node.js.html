Podstawy Node.js

Node.js to środowisko uruchomieniowe JavaScript umożliwiające tworzenie aplikacji po stronie serwera. Pozwala na pisanie kodu w JavaScript na serwerze, co otwiera nowe możliwości dla deweloperów. Poniżej znajdziesz instrukcje instalacji, omówienie podstawowych komend oraz REPL.

Instalacja Node.js

Aby zacząć pracę z Node.js, należy najpierw zainstalować środowisko. Można to zrobić poprzez stronę nodejs.org. Zaleca się pobranie wersji LTS (Long Term Support) dla stabilności.


# Sprawdzenie wersji Node.js po instalacji
node -v

Po instalacji, komenda node -v pozwoli sprawdzić, czy Node.js został poprawnie zainstalowany.

Podstawowe komendy

Node.js oferuje kilka podstawowych komend do zarządzania i testowania kodu. Oto niektóre z nich:

  • node [nazwa_pliku]: Uruchamia dany plik JavaScript za pomocą Node.js.
  • npm init: Inicjuje nowy projekt Node.js, tworząc plik package.json.
  • npm install [nazwa_pakietu]: Instaluje pakiet npm (Node Package Manager) lokalnie do projektu.

# Przykład uruchomienia pliku
node app.js

# Inicjacja nowego projektu
npm init

# Instalacja pakietu np. Express
npm install express

REPL (Read-Eval-Print Loop)

REPL to interaktywna powłoka, która umożliwia wprowadzanie kodu JavaScript, natychmiastowe wykonywanie go i zwracanie wyników. Jest to przydatne narzędzie do testowania małych fragmentów kodu.


# Uruchamianie REPL
node

# Przykład kodu w REPL
> let x = 10;
> x + 5
// Wynik: 15

Aby uruchomić REPL, wystarczy wpisać node w terminalu. Można w nim pisać proste wyrażenia JavaScript i natychmiast otrzymywać wynik.

Zarządzanie wersjami Node.js

Do zarządzania różnymi wersjami Node.js można użyć narzędzia NVM (Node Version Manager). Dzięki niemu możemy łatwo instalować i przełączać się między wersjami Node.js.


# Instalacja NVM
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.3/install.sh | bash

# Sprawdzenie dostępnych wersji Node.js
nvm ls-remote

# Instalacja konkretnej wersji Node.js
nvm install 14.17.0

# Przełączanie się na inną wersję
nvm use 14.17.0

Przykład pokazuje, jak zainstalować NVM i zarządzać wersjami Node.js, co jest przydatne, gdy pracujemy nad różnymi projektami wymagającymi różnych wersji.
