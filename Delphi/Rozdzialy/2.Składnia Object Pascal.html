Składnia Object Pascal

Object Pascal to język programowania używany w Delphi. Poniżej omówimy podstawowe elementy składni: zmienne, typy, klasy, rekordy (records) i interfejsy (interfaces).

Zmienne i typy danych

Zmienne to miejsca w pamięci o nazwie, w których przechowujemy dane.

  • var – słowo kluczowe do deklaracji zmiennej.
  • Typy proste: Integer (liczba całkowita), Real (liczba rzeczywista), String (ciąg znaków).
var
  liczba: Integer;     // zmienna przechowująca liczbę całkowitą
  tekst: String;       // zmienna przechowująca tekst
  wynik: Real;         // zmienna przechowująca liczbę z przecinkiem
begin
  liczba := 10;        // przypisanie wartości do zmiennej
  tekst := 'Witaj';    // przypisanie tekstu
  wynik := 3.14;       // przypisanie liczby rzeczywistej
end;

Klasy

Klasa (class) to szablon, według którego tworzymy obiekty. Zawiera pola (zmienne) i metody (procedury/funkcje).

type
  TOsoba = class
  private
    FImie: String;       // pole prywatne
  public
    constructor Create(const AImie: String); // konstruktor
    procedure PrzedstawSie;                  // metoda
  end;

constructor TOsoba.Create(const AImie: String);
begin
  FImie := AImie;
end;

procedure TOsoba.PrzedstawSie;
begin
  ShowMessage('Cześć, jestem ' + FImie);
end;

var
  osoba: TOsoba;
begin
  osoba := TOsoba.Create('Jan');  // tworzenie obiektu
  osoba.PrzedstawSie;              // wywołanie metody
  osoba.Free;                      // zwolnienie obiektu
end;

Rekordy (records)

Rekord (record) to typ złożony, który grupuje różne pola w jedną strukturę.

type
  TPoint = record
    X, Y: Integer;     // pola rekordu
  end;

var
  punkt: TPoint;
begin
  punkt.X := 10;      // dostęp do pól
  punkt.Y := 20;
  ShowMessage('X=' + IntToStr(punkt.X) + ', Y=' + IntToStr(punkt.Y));
end;

Interfejsy (interfaces)

Interfejs (interface) definiuje zestaw metod, które klasa musi implementować.

type
  IZwierze = interface
    ['{D4A1F8C2-1A3B-4E8C-9A3E-123456789ABC}']    // unikalny identyfikator
    procedure DajGlos;                            // metoda
  end;

  TPies = class(TInterfacedObject, IZwierze)
  public
    procedure DajGlos;                           // implementacja
  end;

procedure TPies.DajGlos;
begin
  ShowMessage('Hau hau!');
end;

var
  zw: IZwierze;
begin
  zw := TPies.Create;    // utworzenie instancji klasy zgodnej z interfejsem
  zw.DajGlos;            // wywołanie metody z interfejsu
end;

Wskazówki

  • Używaj czytelnych nazw zmiennych i typów (np. FImie dla pola klasy).
  • Zwalniaj obiekty za pomocą Free lub korzystaj z try…finally.
  • Rekordy są szybsze od klas, bo przechowywane na stosie.
  • Interfejsy ułatwiają zarządzanie pamięcią dzięki automatycznemu odliczaniu referencji.
  • Stosuj komentarze pod linijkami kodu, by wyjaśniać znaczenie każdego elementu.
