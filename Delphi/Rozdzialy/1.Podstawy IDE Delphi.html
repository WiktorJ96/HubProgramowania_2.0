Podstawy IDE Delphi

Delphi to zintegrowane środowisko programistyczne (IDE) RAD Studio, które umożliwia tworzenie aplikacji w języku Object Pascal (Delphi) za pomocą komponentów wizualnych i szybkiego projektowania. Poniżej omówimy podstawowe elementy IDE i strukturę projektu.

Środowisko RAD Studio

  • Edytor kodu – piszesz tu kod źródłowy plików .pas i .dfm (formy).
  • Form Designer – projektant formularzy, przeciągasz komponenty z palety na formę.
  • Object Inspector – ustawiasz właściwości (properties) i zdarzenia (events) komponentów.
  • Project Manager – widok struktury projektu: pliki, grupy, konfiguracje kompilacji (Debug/Release).
  • Tool Palette – biblioteka komponentów VCL i FMX do szybkiego tworzenia interfejsu.
  • Message/Output Window – wyświetla informacje o kompilacji, błędy i logi.

Struktura projektu Delphi

Standardowy projekt Delphi składa się z pliku projektu .dproj oraz folderu zawierającego moduły (units) i formy:

  • Project1.dproj – główny plik projektu XML z ustawieniami kompilacji.
  • Project1.dpr – plik główny project source, uruchamia aplikację.
  • Unit1.pas i Unit1.dfm – moduł z definicją formularza i plikiem formy.
  • Unit2.pas itd. – kolejne moduły z kodem pomocniczym lub formularzami.
// Przykład pliku projektu: Project1.dpr
program Project1;

uses
  Vcl.Forms,
  Unit1 in 'Unit1.pas' {Form1};

{$R *.res}

begin
  Application.Initialize;
  Application.MainFormOnTaskbar := True;
  Application.CreateForm(TForm1, Form1);
  Application.Run;
end.

Units (moduły)

Unit to plik .pas zawierający kod Object Pascal. Składa się z sekcji interface i implementation:

unit Unit1;

interface

uses
  System.Classes, Vcl.Forms;

type
  TForm1 = class(TForm)
    // deklaracja komponentów, np. Button1: TButton;
  private
    // prywatne pola i metody
  public
    // publiczne metody i właściwości
  end;

var
  Form1: TForm1;

implementation

{$R *.dfm}

// implementacja metod formularza

end.

Forms (formy)

Każda forma ma dwa powiązane pliki:

  • .pas – kod jednostki, deklaracje klasy formularza.
  • .dfm – plik binarny (lub tekstowy) z definicją rozmieszczenia komponentów.
object Form1: TForm1
  Left = 0
  Top = 0
  Caption = 'Form1'
  ClientHeight = 300
  ClientWidth = 400
  object Button1: TButton
    Left = 150
    Top = 130
    Width = 100
    Height = 30
    Caption = 'Kliknij'
    OnClick = Button1Click
  end
end

Wskazówki

  • Zacznij od zrozumienia pliku .dpr – to punkt startowy aplikacji.
  • Unit to enkapsulacja kodu: jedna jednostka – jedna odpowiedzialność.
  • Formy traktuj jak widoki – przełączaj się między edycją kodu a projektantem.
  • Używaj Object Inspector do zmiany właściwości zamiast ręcznie edytować .dfm.
  • Przetestuj kompilację w konfiguracji Debug i Release, by uniknąć niespodzianek.
