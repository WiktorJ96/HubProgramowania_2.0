Debugowanie i Testy

Delphi IDE oferuje potężne narzędzia do debugowania, a także wsparcie dla testów jednostkowych za pomocą DUnitX. Można także profilować aplikację przy pomocy zewnętrznych profilerów lub wbudowanych narzędzi.

Debugger IDE

  • Breakpoints – ustawienie punktu przerwania (klik na marginesie lub F5).
  • Watch List – obserwacja wartości zmiennych i wyrażeń.
  • Call Stack – śledzenie stosu wywołań metod.
  • Evaluate/Modify (Ctrl+F7) – ocena i modyfikacja zmiennych w czasie debugowania.
  • Step Into (F7) / Step Over (F8) / Run to Cursor (Ctrl+F9) – kontrola przepływu wykonania.

DUnitX

Framework do testów jednostkowych w Delphi:

// Przykładowy test DUnitX
uses DUnitX.TestFramework;

type
  [TestFixture]
  TMyTests = class
  public
    [Test]
    procedure TestAdd;
  end;

procedure TMyTests.TestAdd;
begin
  Assert.AreEqual(5, 2+3, '2+3 powinno być 5');
end;

initialization
  TDUnitX.RegisterTestFixture(TMyTests);

Profile narzędzi

  • Insight – profiler CPU dla Delphi (płatny).
  • Sampling Profiler – wbudowane narzędzie w RAD Studio.
  • FastMM4 – kompatybilny profiler pamięci i leak-detection.

Wskazówki

  • Ustawiaj breakpointy tylko w krytycznych fragmentach, by nie spowalniać debugowania.
  • Testuj kluczowe metody przy użyciu DUnitX i automatyzuj uruchamianie testów w CI.
  • Profiluj w trybie release z informacjami debug, by uzyskać wiarygodne dane.
  • Użyj FastMM4 z opcją wyłapywania wycieków pamięci (FullDebugMode).
