Lista Komend SQL

SQL (Structured Query Language) to język zapytań służący do zarządzania relacyjnymi bazami danych. Poniżej znajduje się lista najważniejszych komend SQL wraz z krótkimi opisami ich zastosowania.

  • SELECT – wybiera dane z tabeli.
  • FROM – określa tabelę, z której pobierane są dane.
  • WHERE – filtruje dane na podstawie warunku.
  • INSERT INTO – dodaje nowe dane do tabeli.
  • UPDATE – aktualizuje istniejące dane w tabeli.
  • DELETE – usuwa dane z tabeli.
  • ORDER BY – sortuje wyniki według kolumny (rosnąco lub malejąco).
  • LIMIT – ogranicza liczbę zwracanych wierszy.
  • JOIN – łączy dane z wielu tabel.
    • INNER JOIN – tylko dopasowane wiersze w obu tabelach.
    • LEFT JOIN – wszystkie wiersze z lewej tabeli + dopasowane z prawej.
    • RIGHT JOIN – wszystkie z prawej tabeli + dopasowane z lewej.
    • FULL JOIN – wszystkie wiersze z obu tabel.
  • GROUP BY – grupuje dane na podstawie kolumny.
  • HAVING – filtruje grupy utworzone przez GROUP BY.
  • AS – tworzy alias dla kolumny lub tabeli.
  • IN – sprawdza, czy wartość znajduje się w podanym zestawie.
  • BETWEEN – sprawdza, czy wartość mieści się w podanym zakresie.
  • LIKE – wyszukiwanie według wzorca (np. 'A%').
  • IS NULL / IS NOT NULL – sprawdza czy wartość jest (lub nie jest) NULL.
  • DISTINCT – usuwa duplikaty z wyników.
  • CREATE TABLE – tworzy nową tabelę.
  • ALTER TABLE – zmienia strukturę istniejącej tabeli.
  • DROP TABLE – usuwa tabelę.
  • TRUNCATE TABLE – usuwa wszystkie wiersze z tabeli, ale zostawia jej strukturę.
  • PRIMARY KEY – ustala główny klucz tabeli.
  • FOREIGN KEY – ustala klucz obcy, który wskazuje na inną tabelę.
  • UNIQUE – zapewnia unikalność wartości w kolumnie.
  • DEFAULT – ustawia domyślną wartość kolumny.
  • CHECK – ograniczenie warunków dla wartości w kolumnie.
  • INDEX – tworzy indeks w celu przyspieszenia zapytań.
  • VIEW – tworzy widok (wirtualną tabelę na podstawie zapytania SELECT).
  • BEGIN TRANSACTION / COMMIT / ROLLBACK – zarządzanie transakcjami.

Przykłady użycia kombinacji komend

-- Przykład 1: Pobranie danych pracowników z działu HR, posortowanych według nazwiska
SELECT first_name, last_name
FROM employees
WHERE department = 'HR'
ORDER BY last_name ASC;
-- Użyto: SELECT, FROM, WHERE, ORDER BY
-- Zastosowanie: Kiedy chcesz wyświetlić dane tylko z konkretnego działu i mieć je uporządkowane alfabetycznie
-- Przykład 2: Średnie wynagrodzenie w każdym dziale, tylko jeśli liczba pracowników > 5
SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING COUNT(*) > 5;
-- Użyto: SELECT, GROUP BY, HAVING, funkcji agregujących
-- Zastosowanie: Do analizy danych zbiorczych, np. płac w dużych działach
-- Przykład 3: Połączenie tabel employees i departments, by pokazać nazwiska i nazwy działów
SELECT e.last_name, d.name AS department_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;
-- Użyto: SELECT, JOIN, aliasów AS
-- Zastosowanie: Gdy dane są rozdzielone w różnych tabelach i trzeba je powiązać
-- Przykład 4: Aktualizacja adresu e-mail tylko dla użytkowników z Polski
UPDATE users
SET email = 'nowy@email.pl'
WHERE country = 'Polska';
-- Użyto: UPDATE, SET, WHERE
-- Zastosowanie: Kiedy trzeba zmienić dane tylko dla konkretnej grupy użytkowników
-- Przykład 5: Usunięcie kont użytkowników nieaktywnych przez ponad 2 lata
DELETE FROM users
WHERE last_login < NOW() - INTERVAL 2 YEAR;
-- Użyto: DELETE, WHERE, funkcji daty
-- Zastosowanie: Do czyszczenia nieaktualnych danych
