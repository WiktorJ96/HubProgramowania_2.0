Replikacja w MongoDB

Replikacja to proces utrzymywania wielu kopii danych na różnych serwerach w celu zwiększenia dostępności i niezawodności systemu. MongoDB używa replik setów (ang. replica sets), które zapewniają automatyczne przełączanie awaryjne i redundancję danych.

Konfiguracja Replik Setów

Replik set w MongoDB składa się z jednego podstawowego (ang. primary) i co najmniej jednego lub więcej replik drugorzędnych (ang. secondary). Aby skonfigurować repliki, wykonaj poniższe kroki:


# Uruchomienie MongoDB jako członka replika setu
mongod --replSet \"mójReplSet\" --port 27017 --dbpath /data/db1 --bind_ip localhost

Następnie, po uruchomieniu instancji, skonfiguruj replik set w Mongo Shell:


// Inicjalizacja replika setu
rs.initiate({
    _id: \"mójReplSet\",
    members: [
        { _id: 0, host: \"localhost:27017\" },
        { _id: 1, host: \"localhost:27018\" },
        { _id: 2, host: \"localhost:27019\" }
    ]
});

Zarządzanie Replikacją Danych

Podstawowe operacje związane z zarządzaniem replikacją danych obejmują:

  • Monitorowanie stanu replik setu: Użyj rs.status(), aby sprawdzić aktualny stan repliki.
  • Dodawanie nowych członków: Nowe instancje można dodać za pomocą rs.add(). Przykład:

rs.add(\"localhost:27020\");
  • Usuwanie członków: Użyj rs.remove(), aby usunąć członka z replik setu. Przykład:

rs.remove(\"localhost:27020\");

Automatyczne Przełączanie Awaryjne

W przypadku awarii instancji podstawowej jeden z członków drugorzędnych automatycznie przejmuje rolę podstawową, zapewniając ciągłość działania systemu. Replik set obsługuje to za pomocą algorytmu wyboru lidera.

Podsumowanie

  • Replikacja w MongoDB zapewnia wysoką dostępność danych poprzez replik sety.
  • Replik sety umożliwiają automatyczne przełączanie awaryjne i redundancję danych.
  • Można zarządzać replikacją za pomocą komend rs.initiate(), rs.add(), rs.status(), i rs.remove().

Replikacja

Proces utrzymywania wielu kopii danych na różnych serwerach w celu zwiększenia dostępności i niezawodności systemu. MongoDB używa replik setów, które zapewniają automatyczne przełączanie awaryjne i redundancję danych.

Replica Set (Zestaw replik)

Grupa serwerów MongoDB składająca się z jednego podstawowego (primary) i co najmniej jednego lub więcej replik drugorzędnych (secondary), która utrzymuje ten sam zestaw danych.

Primary Node (Węzeł podstawowy)

Główny węzeł w replica set, który jako jedyny przyjmuje operacje zapisu i domyślnie obsługuje wszystkie operacje odczytu.

Secondary Node (Węzeł drugorzędny)

Węzeł w replica set, który utrzymuje kopię danych z węzła podstawowego. Może obsługiwać operacje odczytu, ale nie może przyjmować operacji zapisu.

Failover (Przełączanie awaryjne)

Automatyczny proces, w którym w przypadku awarii węzła podstawowego jeden z węzłów drugorzędnych przejmuje rolę podstawowego, zapewniając ciągłość działania systemu.
