Funkcje i Wyrażenia Funkcyjne w JavaScript

Funkcje w JavaScript mogą być tworzone nie tylko jako deklaracje, ale także jako wyrażenia. Wyrażenia funkcyjne to funkcje przypisywane do zmiennych, które mogą być anonimowe (bez nazwy) lub nazwane. Pozwalają na bardziej dynamiczne operacje na funkcjach i są często wykorzystywane w programowaniu funkcyjnym.

Wyrażenia funkcyjne

Wyrażenia funkcyjne to funkcje, które są definiowane w miejscu i przypisywane do zmiennej. W przeciwieństwie do funkcji zadeklarowanych za pomocą słowa kluczowego function, wyrażenia funkcyjne nie muszą mieć nazwy i są często używane do tworzenia funkcji anonimowych.

Przykład wyrażenia funkcyjnego


// Tworzenie funkcji jako wyrażenia funkcyjnego
const greet = function(name) {
    return 'Cześć, ' + name + '!';
};

console.log(greet('Kasia')); // Cześć, Kasia!

W powyższym przykładzie funkcja jest przypisana do zmiennej greet. Zamiast deklaracji funkcji, mamy wyrażenie funkcyjne, które tworzy funkcję i przypisuje ją do zmiennej.

Anonimowe wyrażenia funkcyjne

Wyrażenia funkcyjne mogą być anonimowe, co oznacza, że funkcja nie ma nazwy. Anonimowe wyrażenia funkcyjne są szczególnie przydatne, gdy chcemy przekazać funkcję jako argument do innej funkcji.

Przykład anonimowego wyrażenia funkcyjnego


setTimeout(function() {
    console.log('To jest anonimowa funkcja!');
}, 1000);

Funkcja w powyższym przykładzie jest anonimowa, ponieważ nie ma nazwy. Jest przekazywana jako argument do funkcji setTimeout, która wywołuje ją po 1 sekundzie.

Różnica między deklaracją funkcji a wyrażeniem funkcyjnym

Główna różnica między deklaracją funkcji a wyrażeniem funkcyjnym polega na tym, jak te funkcje są hoistowane. Funkcje zadeklarowane za pomocą function są hoistowane, co oznacza, że mogą być wywoływane przed ich deklaracją w kodzie. Wyrażenia funkcyjne nie są hoistowane, więc muszą być zadeklarowane przed ich użyciem.

Przykład hoistingu


// Deklaracja funkcji - działa nawet przed deklaracją
console.log(sayHello('Jan')); // Cześć, Jan!

function sayHello(name) {
    return 'Cześć, ' + name + '!';
}

// Wyrażenie funkcyjne - nie zadziała przed deklaracją
console.log(sayGoodbye('Ania')); // Błąd: Cannot access 'sayGoodbye' before initialization

const sayGoodbye = function(name) {
    return 'Do widzenia, ' + name + '!';
};

W powyższym przykładzie funkcja sayHello może być wywoływana przed jej deklaracją, ponieważ jest hoistowana. Natomiast wyrażenie funkcyjne sayGoodbye nie jest hoistowane, więc musi być zadeklarowane przed użyciem.

Wyrażenia funkcyjne nazwane

Wyrażenia funkcyjne mogą mieć nazwę, co może być przydatne podczas debugowania lub w rekurencji.

Przykład nazwanych wyrażeń funkcyjnych


const factorial = function fact(n) {
    if (n <= 1) return 1;
    return n * fact(n - 1); // Rekurencja z użyciem nazwy funkcji
};

console.log(factorial(5)); // 120

W tym przykładzie funkcja fact jest wyrażeniem funkcyjnym nazwanym. Umożliwia to rekurencyjne wywoływanie funkcji w jej własnym ciele.

Podsumowanie

  • Wyrażenia funkcyjne pozwalają na przypisywanie funkcji do zmiennych lub przekazywanie ich jako argumenty do innych funkcji.
  • Funkcje anonimowe nie mają nazwy i są często używane w sytuacjach, gdzie funkcja jest przekazywana jako argument.
  • W przeciwieństwie do deklaracji funkcji, wyrażenia funkcyjne nie są hoistowane, więc muszą być zadeklarowane przed użyciem.
  • Wyrażenia funkcyjne mogą być nazwane, co jest przydatne w rekurencji i debugowaniu.
