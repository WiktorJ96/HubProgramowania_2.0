Instrukcje Warunkowe i Pętle

Ruby oferuje różnorodne konstrukcje sterujące przepływem programu, takie jak instrukcje warunkowe oraz pętle, które pozwalają na wykonywanie określonych bloków kodu w zależności od warunków lub liczby powtórzeń.

Instrukcje warunkowe

Instrukcje warunkowe pozwalają na wykonywanie fragmentów kodu w zależności od spełnienia określonych warunków. Najczęściej używane to if, else, elsif oraz unless.

if, else, elsif

Instrukcja if wykonuje blok kodu, jeśli warunek jest prawdziwy (true). Można dodać elsif oraz else dla dodatkowych warunków.


        x = 5

        if x > 10
          puts "x jest większe niż 10"
        elsif x == 5
          puts "x jest równe 5"
        else
          puts "x jest mniejsze niż 10"
        end

unless

Instrukcja unless działa odwrotnie do if. Wykonuje kod, jeśli warunek jest fałszywy (false).


        y = 3

        unless y > 5
          puts "y jest mniejsze lub równe 5"
        end

Pętle

Pętle służą do powtarzania określonych operacji. Ruby obsługuje różne rodzaje pętli, takie jak while, until, for oraz pętle iteracyjne jak each.

while

Pętla while wykonuje blok kodu, dopóki warunek jest prawdziwy (true).


        i = 0

        while i < 5
          puts "i = #{i}"
          i += 1
        end

until

Pętla until działa odwrotnie do while. Wykonuje kod, dopóki warunek jest fałszywy (false).


        j = 0

        until j > 5
          puts "j = #{j}"
          j += 1
        end

for

Pętla for iteruje po zakresie lub tablicy.


        for k in 1..5
          puts "k = #{k}"
        end

each

Metoda each pozwala na iterację po elementach kolekcji, takich jak tablice.


        fruits = ["apple", "banana", "cherry"]

        fruits.each do |fruit|
          puts "Owoc: #{fruit}"
        end

Pętla loop

Pętla loop jest nieskończoną pętlą, dopóki nie zostanie przerwana przez break.


        count = 0

        loop do
          puts "count = #{count}"
          count += 1
          break if count > 5
        end

Przerywanie pętli: break, next

  • break: Przerywa działanie pętli przed jej zakończeniem (np. zakończenie nieskończonej pętli).
  • next: Przeskakuje do kolejnej iteracji pętli, pomijając resztę kodu w bieżącej iteracji.

        (1..10).each do |n|
          next if n % 2 == 0  # Pomija liczby parzyste
          puts n
        end
