Workflow i Statusy

W Jira workflow to schemat, który definiuje żywotność zadania – od momentu jego utworzenia, poprzez kolejne etapy, aż do zakończenia. Dzięki niemu można precyzyjnie kontrolować, w jakim stanie znajduje się każde zadanie oraz jakie zasady muszą być spełnione przy zmianie statusu.

Definiowanie Statusów

Statusy określają aktualny stan zadania. Typowymi statusami są:

  • To Do – zadanie do wykonania
  • In Progress – zadanie w trakcie realizacji
  • Code Review – zadanie oczekujące na przegląd kodu
  • Done – zadanie zakończone

Możesz definiować dowolną liczbę statusów, dostosowując je do specyfiki projektu i procesu pracy.

Przejścia między Statusami

Przejścia (transitions) określają, jak zadanie zmienia swój status. Każde przejście łączy jeden status z kolejnym, na przykład:

  • Start Work: zmiana statusu z To Do na In Progress
  • Submit for Review: zmiana statusu z In Progress na Code Review
  • Mark as Done: zmiana statusu z Code Review na Done

Przejścia umożliwiają kontrolowanie przepływu zadań oraz określają, kto i kiedy może dokonać zmiany statusu.

Konfiguracja Warunków i Reguł

Aby przejścia były skuteczne i zgodne z wymogami procesu, Jira pozwala na dodanie:

  • Warunków (Conditions): Określających, kto może wykonać przejście, np. tylko użytkownik z rolą Developer.
  • Walidatorów (Validators): Sprawdzających, czy wymagane pola (np. Description) są wypełnione przed zatwierdzeniem zmiany statusu.
  • Funkcji po przejściu (Post Functions): Automatyzujących działania po wykonaniu przejścia, np. przypisanie zadania do bieżącego użytkownika czy wysłanie powiadomienia.

Dzięki tym ustawieniom workflow w Jira staje się narzędziem wspierającym efektywne zarządzanie procesem pracy, eliminując błędy i usprawniając komunikację w zespole.

Przykładowa Konfiguracja Przejścia

Poniżej znajduje się przykładowy zapis konfiguracji przejścia w formacie JSON, ilustrujący najważniejsze elementy:


{
  "transition": "Start Work",
  "from": "To Do",
  "to": "In Progress",
  "conditions": [
    { "role": "Developer" }
  ],
  "validators": [
    { "field_required": "Description" }
  ],
  "post_functions": [
    { "assign_to": "currentUser" },
    { "send_notification": true }
  ]
}

W powyższym przykładzie:

  • Przejście "Start Work" zmienia status zadania z "To Do" na "In Progress".
  • Warunkiem jest, aby użytkownik należał do roli Developer.
  • Walidator sprawdza, czy pole Description nie jest puste.
  • Funkcje po przejściu automatycznie przypisują zadanie do aktualnego użytkownika i wysyłają powiadomienie.

Dzięki tak skonfigurowanemu workflow możesz mieć pewność, że przejścia w Twoim projekcie są bezpieczne i zgodne z ustalonymi zasadami.
