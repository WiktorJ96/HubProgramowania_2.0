Uruchamianie testów w Jasmine

Jasmine oferuje elastyczne opcje uruchamiania testów, zarówno w przeglądarce, jak i w konsoli. Poniżej opisano proces konfiguracji i uruchamiania testów w obu środowiskach.

Konfiguracja testów w przeglądarce

Testy Jasmine można uruchamiać w przeglądarce, korzystając z wygenerowanego interfejsu HTML. Poniżej przedstawiono kroki konfiguracji:

1. Instalacja Jasmine


npm install --save-dev jasmine

2. Inicjalizacja projektu


npx jasmine init

Powyższe polecenie utworzy pliki i foldery potrzebne do uruchamiania testów, w tym spec/support/jasmine.json.

3. Dodanie pliku HTML dla testów

Utwórz plik HTML, który załaduje Jasmine i pliki testowe:





    
    Testy Jasmine

Otwórz ten plik w przeglądarce, aby uruchomić testy i zobaczyć wyniki w graficznym interfejsie Jasmine.

Użycie Jasmine CLI do uruchamiania w konsoli

Jasmine CLI umożliwia uruchamianie testów w konsoli, co jest szczególnie przydatne w zautomatyzowanych środowiskach.

1. Instalacja Jasmine CLI


npm install --global jasmine

2. Uruchamianie testów

Aby uruchomić testy, użyj polecenia:


jasmine

Możesz także uruchomić konkretne pliki testowe, podając ich ścieżkę:


jasmine spec/mySpec.js

3. Konfiguracja pliku jasmine.json

Plik jasmine.json pozwala dostosować uruchamianie testów. Przykład konfiguracji:


{
  "spec_dir": "spec",
  "spec_files": [
    "**/*[sS]pec.js"
  ],
  "helpers": [
    "helpers/**/*.js"
  ],
  "random": true,
  "stopSpecOnExpectationFailure": false
}

Możesz zmienić kolejność testów na losową (random) lub ustawić, czy testy mają się zatrzymać przy pierwszym błędzie (stopSpecOnExpectationFailure).

Porównanie przeglądarki i CLI

Środowisko Zalety Wady
Przeglądarka
  • Intuicyjny graficzny interfejs.
  • Łatwe w debugowaniu dzięki narzędziom deweloperskim przeglądarki.
  • Wymaga ręcznego otwierania pliku HTML.
  • Mniej odpowiednie dla CI/CD.
CLI
  • Automatyzacja w CI/CD.
  • Szybkie i wydajne uruchamianie testów.
  • Brak graficznego interfejsu.
  • Trudniejsze debugowanie.

Podsumowanie

Jasmine oferuje dwie wygodne metody uruchamiania testów:

  • Konfiguracja testów w przeglądarce dla łatwego debugowania i intuicyjnego interfejsu.
  • Użycie CLI do szybkiego i zautomatyzowanego uruchamiania testów w konsoli.

Wybór zależy od potrzeb projektu i środowiska, w którym testy są wykonywane.
