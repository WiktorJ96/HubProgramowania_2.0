Podstawy Jasmine

Jasmine to popularne narzędzie do testowania aplikacji JavaScript. Jest to framework umożliwiający pisanie testów jednostkowych (unit tests) w prosty i przejrzysty sposób. W tej sekcji przedstawimy konfigurację, uruchamianie testów oraz podstawowe struktury takie jak describe i it.

Konfiguracja

Aby rozpocząć korzystanie z Jasmine, musisz go zainstalować za pomocą menedżera pakietów npm:


npm install --save-dev jasmine

Następnie skonfiguruj Jasmine, uruchamiając polecenie inicjalizujące:


npx jasmine init

Powyższe polecenie utworzy odpowiednią strukturę katalogów i pliki potrzebne do uruchamiania testów.

Uruchamianie testów

Testy w Jasmine można uruchamiać za pomocą polecenia:


npx jasmine

To polecenie automatycznie znajdzie pliki z testami w katalogu spec i uruchomi je.

Podstawowe struktury

Jasmine używa struktur takich jak describe i it do organizowania i definiowania testów.

Przykład

Poniżej przedstawiono prosty przykład testu w Jasmine:


describe('Prosty test', () => {
    it('powinien dodać dwie liczby', () => {
        const suma = 2 + 3;
        expect(suma).toBe(5);
        // Oczekujemy, że wynik dodawania 2 i 3 wynosi 5
    });
});

describe jest używane do grupowania testów w zestawy, a it definiuje pojedynczy test. Oczekiwania (expectations) są definiowane za pomocą funkcji expect.

Podsumowanie

Jasmine jest prostym i efektywnym narzędziem do testowania aplikacji JavaScript. Warto pamiętać o podstawowych krokach:

  • Instalacja i konfiguracja Jasmine
  • Uruchamianie testów za pomocą npx jasmine
  • Organizowanie testów przy użyciu describe i it
