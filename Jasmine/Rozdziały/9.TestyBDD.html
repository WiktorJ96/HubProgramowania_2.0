Testy BDD w Jasmine

Testowanie w stylu BDD (Behavior-Driven Development) skupia się na opisywaniu zachowań aplikacji w sposób zrozumiały dla programistów i interesariuszy. Jasmine, jako framework BDD, pozwala pisać czytelne testy za pomocą opisowych metod, takich jak describe, it, i expect.

Elementy stylu BDD w Jasmine

Styl BDD w Jasmine opiera się na następujących elementach:

  • describe: Grupa testów opisująca moduł lub funkcję.
  • it: Pojedynczy przypadek testowy opisujący konkretne zachowanie.
  • expect: Wyrażenie, które sprawdza oczekiwane zachowanie.

Przykład prostego testu w stylu BDD


describe('Kalkulator', () => {
    it('powinien dodać dwie liczby', () => {
        const suma = 2 + 3;
        expect(suma).toBe(5);
        // Sprawdza, czy dodanie 2 i 3 daje wynik 5
    });

    it('powinien pomnożyć dwie liczby', () => {
        const iloczyn = 4 * 3;
        expect(iloczyn).toBe(12);
        // Sprawdza, czy mnożenie 4 i 3 daje wynik 12
    });
});

Opisowe testy

Testy w stylu BDD powinny być zrozumiałe i jasno komunikować oczekiwane zachowanie aplikacji. Stosowanie pełnych zdań w describe i it poprawia czytelność.

Przykład testu z czytelnym opisem


describe('Funkcja zamówienia produktu', () => {
    it('powinna dodać produkt do koszyka, jeśli jest dostępny', () => {
        const koszyk = [];
        const produkt = { id: 1, nazwa: 'Laptop', dostepny: true };

        if (produkt.dostepny) {
            koszyk.push(produkt);
        }

        expect(koszyk.length).toBe(1);
        expect(koszyk[0].nazwa).toBe('Laptop');
        // Sprawdza, czy produkt został dodany do koszyka
    });

    it('nie powinna dodać produktu do koszyka, jeśli jest niedostępny', () => {
        const koszyk = [];
        const produkt = { id: 2, nazwa: 'Telefon', dostepny: false };

        if (produkt.dostepny) {
            koszyk.push(produkt);
        }

        expect(koszyk.length).toBe(0);
        // Sprawdza, czy koszyk pozostał pusty
    });
});

Organizacja testów w stylu BDD

W stylu BDD testy są organizowane hierarchicznie, aby jasno pokazać zależności między modułami i zachowaniami. Przykład:


describe('Moduł logowania', () => {
    describe('Kiedy użytkownik wprowadzi poprawne dane', () => {
        it('powinien zalogować użytkownika', () => {
            const isLoggedIn = true;
            expect(isLoggedIn).toBe(true);
        });
    });

    describe('Kiedy użytkownik wprowadzi błędne dane', () => {
        it('powinien wyświetlić komunikat o błędzie', () => {
            const komunikat = 'Nieprawidłowe dane logowania';
            expect(komunikat).toBe('Nieprawidłowe dane logowania');
        });
    });
});

Taka struktura testów poprawia czytelność i ułatwia zrozumienie zachowania aplikacji.

Podsumowanie

Testy BDD w Jasmine pomagają tworzyć czytelne i zrozumiałe testy:

  • Opisowe describe i it komunikują oczekiwane zachowanie aplikacji.
  • Wyrażenia expect sprawdzają wyniki testów.
  • Hierarchia testów pozwala na jasne grupowanie zachowań.

Styl BDD wspiera współpracę między programistami, testerami i interesariuszami, ułatwiając wspólne zrozumienie wymagań.
