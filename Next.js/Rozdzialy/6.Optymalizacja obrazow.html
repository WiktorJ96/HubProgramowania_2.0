Optymalizacja Obrazów w Next.js

Next.js oferuje wbudowaną optymalizację obrazów za pomocą komponentu Image. Dzięki temu obrazy są automatycznie optymalizowane pod kątem rozmiaru, formatu oraz ładowania.

Komponent Image

Komponent Image jest częścią Next.js i zapewnia automatyczną optymalizację obrazów, takich jak lazy loading (leniwie ładowanie), różne formaty obrazów (np. WebP) oraz responsywne rozmiary. Przykład użycia:

Ładowanie obrazów z zewnętrznych źródeł

Aby ładować obrazy z zewnętrznych domen, należy skonfigurować next.config.js. Przykład konfiguracji:


// next.config.js
module.exports = {
  images: {
    domains: ['example.com'], // Dodaj domeny, z których chcesz ładować obrazy
  },
};

Responsywne rozmiary obrazów

Next.js umożliwia łatwe ustawienie responsywnych obrazów za pomocą parametru layout. Możliwe wartości to:

  • fixed: Stały rozmiar obrazu (domyślne ustawienie).
  • intrinsic: Obraz zachowuje swoje proporcje.
  • responsive: Obraz dopasowuje się do rozmiaru rodzica.
  • fill: Obraz wypełnia całą przestrzeń rodzica.

import Image from 'next/image';

const ResponsiveImage = () => {
  return (
    <div style={{ width: '100%', maxWidth: '600px' }}>
      <Image 
        src="/responsive-example.jpg" 
        alt="Obraz responsywny" 
        layout="responsive" 
        width={16} 
        height={9} 
      />
    </div>
  );
};

export default ResponsiveImage;

W przypadku layoutu responsive, wartości width i height definiują proporcje obrazu (np. 16:9).

Lazy Loading

Wszystkie obrazy w Next.js domyślnie korzystają z lazy loading, co oznacza, że obraz jest ładowany tylko wtedy, gdy użytkownik przewinie stronę do jego lokalizacji. Nie wymaga to dodatkowej konfiguracji.

Optymalizacja obrazów SVG

Dla obrazów SVG zaleca się stosowanie komponentów lub bezpośredniego osadzania w kodzie. Komponent Image nie optymalizuje plików SVG, ponieważ są one już zoptymalizowane.

Podsumowanie

Next.js ułatwia optymalizację obrazów, zapewniając lepszą wydajność i doświadczenie użytkownika. Dzięki komponentowi Image można automatycznie ładować obrazy responsywnie, korzystać z lazy loading oraz optymalizować obrazy z zewnętrznych źródeł. Więcej informacji znajdziesz w dokumentacji Next.js.
