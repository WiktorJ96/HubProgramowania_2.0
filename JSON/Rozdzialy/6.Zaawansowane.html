Zaawansowane Koncepcje JSON

JSON Schema

JSON Schema to sposób definiowania struktury i walidacji dokumentów JSON. Pozwala określić, jakie typy danych i formaty są akceptowane, co ułatwia walidację i zapewnia zgodność danych w dużych systemach.

Podstawowy przykład JSON Schema:

{
    "$schema": "http://json-schema.org/draft-07/schema#",
    "title": "User",
    "type": "object",
    "properties": {
        "id": { "type": "integer" },
        "name": { "type": "string" },
        "email": { "type": "string", "format": "email" }
    },
    "required": ["id", "name", "email"]
}

Ten schemat określa, że obiekt JSON musi zawierać pola id, name i email, gdzie id jest liczbą całkowitą, a email musi być w formacie e-mail.

JSON-LD (Linked Data)

JSON-LD (JSON for Linked Data) to format rozszerzający JSON, który umożliwia dodawanie kontekstu do danych, co pomaga zintegrować je z siecią semantyczną. JSON-LD jest szeroko stosowany do oznaczania danych na stronach internetowych, aby umożliwić lepsze rozpoznawanie treści przez wyszukiwarki.

Przykład JSON-LD:

{
    "@context": "https://schema.org",
    "@type": "Person",
    "name": "Alice",
    "jobTitle": "Software Engineer",
    "telephone": "+1-555-555-5555",
    "url": "http://www.example.com/alice"
}

W powyższym przykładzie @context wskazuje na kontekst z schema.org, co pozwala wyszukiwarkom lepiej rozpoznać, że ten obiekt JSON opisuje osobę.

JSON Web Tokens (JWT)

JSON Web Tokens (JWT) to standard służący do bezpiecznego przesyłania informacji między dwiema stronami. JWT są używane głównie do autoryzacji i uwierzytelniania w aplikacjach internetowych, gdzie klient po zalogowaniu otrzymuje token dołączany do kolejnych żądań.

Struktura JWT składa się z trzech części:

  • Header - zawiera typ tokenu (JWT) i algorytm szyfrowania.
  • Payload - zawiera dane, np. identyfikator użytkownika i uprawnienia.
  • Signature - zapewnia integralność danych, zapobiegając ich modyfikacji.

Przykład JWT:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOjEyMywiZW1haWwiOiJhbGljZUBleGFtcGxlLmNvbSJ9.TJVA95OrM7E2cBab30RMHrHDcEfxjoYZgeFONFh7HgQ

JWT jest używany głównie w nagłówkach autoryzacyjnych, np. Authorization: Bearer [token], aby potwierdzić tożsamość użytkownika podczas dostępu do zasobów chronionych.

Zastosowanie zaawansowanych koncepcji JSON

  • JSON Schema - zapewnia walidację i spójność danych w aplikacjach, przydatne do tworzenia złożonych systemów i API.
  • JSON-LD - umożliwia lepsze zrozumienie struktury danych przez wyszukiwarki i integrację z siecią semantyczną.
  • JWT - stosowany do bezpiecznego uwierzytelniania i autoryzacji, zwłaszcza w aplikacjach internetowych.
