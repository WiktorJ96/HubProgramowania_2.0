Struktura i składnia JSON

Obiekty

Obiekt w JSON to zbiór par klucz-wartość, gdzie klucz jest zawsze ciągiem znaków (string) ujętym w podwójne cudzysłowy `"`. Każda para klucz-wartość jest oddzielona dwukropkiem `:`, a kolejne pary oddziela się przecinkiem. Obiekt otoczony jest nawiasami klamrowymi `{ }`.

Przykład obiektu JSON:

{
    "name": "Alice",
    "age": 25,
    "isStudent": false
}

Tablice

Tablica w JSON to lista wartości otoczona nawiasami kwadratowymi `[ ]`. Elementy tablicy są oddzielone przecinkami i mogą przyjmować różne typy danych, takie jak liczby, stringi, obiekty lub inne tablice.

Przykład tablicy JSON:

[
    "Red",
    "Green",
    "Blue"
]

Typy Danych JSON

JSON obsługuje następujące typy danych:

  • String: Ujęty w cudzysłowy, np. `"Hello"`
  • Number: Liczby całkowite lub zmiennoprzecinkowe, np. `42`, `3.14`
  • Boolean: Wartości logiczne, np. `true`, `false`
  • Null: Wartość pusta lub nieznana, zapisana jako `null`
  • Obiekt: Zestaw par klucz-wartość, ujęty w `{ }`
  • Tablica: Lista wartości, ujęta w `[ ]`

Poprawność Składniowa JSON

  • JSON musi być zapisany w formacie UTF-8.
  • Klucze muszą być zawsze ujęte w podwójne cudzysłowy `"`.
  • Obiekty otaczamy nawiasami klamrowymi `{ }`, a tablice nawiasami kwadratowymi `[ ]`.
  • Każda para klucz-wartość w obiekcie musi być oddzielona przecinkiem, ale ostatni element nie powinien mieć przecinka.
  • JSON nie obsługuje komentarzy, więc w plikach JSON nie można dodawać żadnych adnotacji.

Tworzenie Poprawnych Dokumentów JSON

Aby utworzyć poprawny dokument JSON, należy stosować się do zasad składniowych JSON. Ważne jest, aby każdy klucz był ujęty w cudzysłowy, a wartości miały poprawne typy. JSON obsługuje stringi, liczby, boolean, null, obiekty oraz tablice.

Przykład poprawnego dokumentu JSON:

{
    "person": {
        "name": "Alice",
        "age": 25,
        "isStudent": false
    },
    "colors": ["Red", "Green", "Blue"],
    "height": 5.8,
    "nullValue": null,
    "isMember": true
}
