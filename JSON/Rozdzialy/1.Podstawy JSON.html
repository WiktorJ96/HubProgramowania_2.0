Podstawy JSON

Definicja JSON

JSON (JavaScript Object Notation) to lekki format wymiany danych, oparty na składni obiektów JavaScript. Jest używany głównie do przekazywania danych między serwerem a aplikacjami internetowymi lub mobilnymi. JSON jest czytelny dla ludzi i łatwy do przetworzenia przez maszyny.

Cele i Zastosowania JSON

Cele: JSON ma uprościć przesyłanie danych między systemami w sposób zrozumiały i efektywny. Głównym celem było stworzenie formatu lżejszego od XML, który jest łatwy do parsowania (czytania i interpretowania) przez różne języki programowania.

Zastosowania:

  • Przechowywanie i wymiana danych między serwerami a aplikacjami.
  • Przechowywanie konfiguracji aplikacji.
  • Komunikacja między usługami REST API.
  • Przesyłanie danych w aplikacjach internetowych oraz mobilnych.

Zasady Poprawności Składniowej JSON

  • JSON musi być zapisany w formacie UTF-8.
  • Obiekty muszą być ujęte w nawiasy klamrowe `{ }`, a tablice w nawiasy kwadratowe `[ ]`.
  • Klucze muszą być ujęte w cudzysłowy `"`.
  • JSON nie obsługuje komentarzy.
