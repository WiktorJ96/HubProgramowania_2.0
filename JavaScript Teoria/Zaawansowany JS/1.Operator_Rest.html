Wyjaśnienie działania operatora rest

Operator rest (trzy kropki: ...) w JavaScript umożliwia zbieranie wszystkich pozostałych argumentów funkcji lub elementów tablicy/obiektu, które nie zostały przypisane do innych zmiennych. To pozwala na bardziej elastyczne operowanie na zmiennej liczbie argumentów oraz łatwiejszą manipulację danymi.

Przykład w funkcji


        function sum(...numbers) {
            return numbers.reduce((acc, curr) => acc + curr, 0);
        }

        console.log(sum(1, 2, 3, 4)); // 10

W powyższym przykładzie operator rest zbiera wszystkie przekazane argumenty i zamienia je w tablicę numbers. Dzięki temu funkcja może przyjmować dowolną liczbę argumentów.

Przykład w destrukturyzacji obiektu


        const user = { name: 'Ania', age: 30, city: 'Kraków', country: 'Polska' };
        const { name, ...rest } = user;

        console.log(name); // 'Ania'
        console.log(rest); // { age: 30, city: 'Kraków', country: 'Polska' }

W tym przypadku operator rest pozwala na wyodrębnienie wartości name z obiektu, a pozostałe właściwości są zebrane w nowy obiekt rest.

Zastosowanie

Operator rest jest szczególnie przydatny, gdy nie znamy z góry liczby argumentów funkcji lub chcemy pracować z obiektami/tablicami, wyciągając z nich pewne wartości, a resztę przypisując do nowej struktury. Znajduje on również zastosowanie w destrukturyzacji, czyniąc kod bardziej elastycznym i zwięzłym.

  • Umożliwia operacje na dowolnej liczbie argumentów funkcji.
  • Ułatwia destrukturyzację obiektów i tablic, pozwalając na efektywne rozdzielanie danych.
