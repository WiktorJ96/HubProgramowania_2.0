Jak zaimplementować nawigację w aplikacji React

W React do implementacji nawigacji między różnymi widokami aplikacji używa się biblioteki React Router. React Router pozwala na tworzenie wielostronicowych aplikacji poprzez zarządzanie różnymi "trasami" (ang. routes), które odpowiadają różnym adresom URL w aplikacji.

Instalacja React Router

Aby dodać nawigację do aplikacji React, najpierw musisz zainstalować react-router-dom. Możesz to zrobić za pomocą npm lub yarn:


# Instalacja za pomocą npm
npm install react-router-dom

# Instalacja za pomocą yarn
yarn add react-router-dom

Po zainstalowaniu biblioteki, możemy przystąpić do implementacji nawigacji.

Tworzenie podstawowych tras

React Router pozwala definiować różne trasy, które odpowiadają różnym komponentom aplikacji. Oto podstawowy przykład konfiguracji tras za pomocą komponentów BrowserRouter, Route i Switch:


import React from 'react';
import ReactDOM from 'react-dom';
import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';

// Komponenty dla poszczególnych stron
function Home() {
  return <h2>Strona główna</h2>;
}

function About() {
  return <h2>O nas</h2>;
}

function Contact() {
  return <h2>Kontakt</h2>;
}

// Główna aplikacja
function App() {
  return (
    <Router>
      <div>
        <nav>
          <ul>
            <li><a href="/">Strona główna</a></li>
            <li><a href="/about">O nas</a></li>
            <li><a href="/contact">Kontakt</a></li>
          </ul>
        </nav>

        <Switch>
          <Route path="/about">
            <About />
          </Route>
          <Route path="/contact">
            <Contact />
          </Route>
          <Route path="/">
            <Home />
          </Route>
        </Switch>
      </div>
    </Router>
  );
}

ReactDOM.render(<App />, document.getElementById('root'));

W powyższym przykładzie mamy prostą nawigację z trzema trasami: / dla strony głównej, /about dla strony "O nas" oraz /contact dla strony kontaktowej.

Dodawanie linków za pomocą Link

Zamiast używać standardowych linków HTML, React Router udostępnia komponent Link, który umożliwia nawigację bez odświeżania całej strony. Oto przykład użycia Link:


import React from 'react';
import ReactDOM from 'react-dom';
import { BrowserRouter as Router, Route, Switch, Link } from 'react-router-dom';

function Home() {
  return <h2>Strona główna</h2>;
}

function About() {
  return <h2>O nas</h2>;
}

function Contact() {
  return <h2>Kontakt</h2>;
}

function App() {
  return (
    <Router>
      <div>
        <nav>
          <ul>
            <li><Link to="/">Strona główna</Link></li>
            <li><Link to="/about">O nas</Link></li>
            <li><Link to="/contact">Kontakt</Link></li>
          </ul>
        </nav>

        <Switch>
          <Route path="/about">
            <About />
          </Route>
          <Route path="/contact">
            <Contact />
          </Route>
          <Route path="/">
            <Home />
          </Route>
        </Switch>
      </div>
    </Router>
  );
}

ReactDOM.render(<App />, document.getElementById('root'));

Komponent Link zmienia adres URL bez odświeżania strony, co pozwala na płynną nawigację w aplikacji.

Używanie exact do dokładnego dopasowania trasy

Domyślnie, React Router dopasowuje trasę, jeśli pasuje do początkowego fragmentu ścieżki URL. Aby upewnić się, że trasa / (strona główna) jest renderowana tylko wtedy, gdy ścieżka jest dokładnie taka, możesz użyć atrybutu exact:


<Switch>
  <Route path="/" exact>
    <Home />
  </Route>
  <Route path="/about">
    <About />
  </Route>
  <Route path="/contact">
    <Contact />
  </Route>
</Switch>

Użycie exact sprawia, że trasa / będzie renderowana tylko wtedy, gdy ścieżka będzie dokładnie równa /, a nie np. /about lub /contact.

Dynamiczne trasy

React Router pozwala również na definiowanie dynamicznych tras, które zawierają parametry. Oto przykład trasy z dynamicznym parametrem id:


import React from 'react';
import ReactDOM from 'react-dom';
import { BrowserRouter as Router, Route, Switch, Link, useParams } from 'react-router-dom';

function User() {
  const { id } = useParams();
  return <h2>Użytkownik ID: {id}</h2>;
}

function App() {
  return (
    <Router>
      <div>
        <nav>
          <ul>
            <li><Link to="/user/1">Użytkownik 1</Link></li>
            <li><Link to="/user/2">Użytkownik 2</Link></li>
          </ul>
        </nav>

        <Switch>
          <Route path="/user/:id">
            <User />
          </Route>
        </Switch>
      </div>
    </Router>
  );
}

ReactDOM.render(<App />, document.getElementById('root'));

W tym przykładzie używamy useParams, aby uzyskać dostęp do parametru id z trasy /user/:id. Umożliwia to dynamiczne wyświetlanie informacji o użytkownikach w zależności od wartości parametru.

Podsumowanie

Implementacja nawigacji w aplikacji React jest łatwa dzięki React Router. Dzięki tej bibliotece możemy definiować trasy, dynamicznie zmieniać URL bez odświeżania strony oraz tworzyć dynamiczne trasy z parametrami. useParams pozwala na dostęp do zmiennych w trasach, a Link zapewnia płynną nawigację w aplikacji bez przeładowania strony.
