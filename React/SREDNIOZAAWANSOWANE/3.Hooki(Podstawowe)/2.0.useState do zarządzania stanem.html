Używanie useState do zarządzania stanem w React

W nowoczesnych aplikacjach React, funkcjonalne komponenty stały się standardem. Do zarządzania stanem w tych komponentach używamy hooka useState. useState pozwala na dodawanie lokalnego stanu do komponentów funkcyjnych. Z jego pomocą można dynamicznie zarządzać danymi i reagować na interakcje użytkownika.

Podstawowe użycie useState

Hook useState zwraca dwuelementową tablicę: pierwszym elementem jest aktualna wartość stanu, a drugim funkcja do jej aktualizacji. Oto jak wygląda podstawowa implementacja:


import React, { useState } from 'react';

function Counter() {
  // Inicjalizacja stanu z początkową wartością 0
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Kliknięto {count} razy</p>
      <button onClick={() => setCount(count + 1)}>Kliknij mnie</button>
    </div>
  );
}

ReactDOM.render(<Counter />, document.getElementById('root'));

W tym przykładzie stan count jest zainicjalizowany wartością 0 za pomocą useState(0). Funkcja setCount aktualizuje stan, a komponent renderuje się ponownie, gdy stan zostanie zmieniony.

Przykład z wieloma wartościami stanu

W komponentach React możesz używać wielu hooków useState, aby zarządzać różnymi wartościami stanu. Oto przykład z kilkoma wartościami stanu:


import React, { useState } from 'react';

function UserInfo() {
  const [name, setName] = useState('');
  const [age, setAge] = useState(0);

  return (
    <div>
      <div>
        <label>
          Imię: <input type="text" value={name} onChange={(e) => setName(e.target.value)} />
        </label>
      </div>
      <div>
        <label>
          Wiek: <input type="number" value={age} onChange={(e) => setAge(e.target.value)} />
        </label>
      </div>
      <p>Witaj, {name}. Masz {age} lat.</p>
    </div>
  );
}

ReactDOM.render(<UserInfo />, document.getElementById('root'));

W tym przykładzie używamy dwóch wartości stanu: name i age. Każdy z nich jest zarządzany oddzielnie przy użyciu osobnych hooków useState.

Aktualizacja stanu w oparciu o poprzedni stan

Jeśli potrzebujemy zaktualizować stan w oparciu o jego poprzednią wartość, możemy przekazać funkcję jako argument do setState. Oto jak to zrobić:


import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  const increment = () => {
    setCount((prevCount) => prevCount + 1);
  };

  return (
    <div>
      <p>Kliknięto {count} razy</p>
      <button onClick={increment}>Kliknij mnie</button>
    </div>
  );
}

ReactDOM.render(<Counter />, document.getElementById('root'));

Przekazanie funkcji do setCount zapewnia, że najnowsza wartość stanu będzie brana pod uwagę podczas aktualizacji. To przydatne, gdy aktualizacja stanu zależy od jego poprzedniej wartości.

Domyślne wartości stanu

Możemy także ustawić domyślną wartość stanu, która może pochodzić z bardziej skomplikowanych obliczeń. W takim przypadku możemy przekazać funkcję jako argument do useState. Ta funkcja zostanie wykonana tylko raz, przy pierwszym renderowaniu:


import React, { useState } from 'react';

function calculateInitialCount() {
  return 10; // Na przykład, wynik obliczeń
}

function Counter() {
  const [count, setCount] = useState(calculateInitialCount);

  return (
    <div>
      <p>Licznik: {count}</p>
      <button onClick={() => setCount(count + 1)}>Zwiększ licznik</button>
    </div>
  );
}

ReactDOM.render(<Counter />, document.getElementById('root'));

Funkcja calculateInitialCount zostanie wywołana tylko raz, podczas inicjalizacji stanu. To dobre podejście, jeśli musimy przeprowadzić kosztowne obliczenia tylko przy pierwszym renderowaniu komponentu.

Podsumowanie

Hook useState umożliwia zarządzanie lokalnym stanem w komponentach funkcyjnych w React. Jego użycie jest proste i pozwala na dynamiczne aktualizowanie danych w komponentach. Możemy zarządzać wieloma wartościami stanu, aktualizować stan w oparciu o poprzednie wartości oraz ustawiać domyślne wartości stanu za pomocą funkcji. useState jest jednym z najważniejszych hooków w React, umożliwiając budowanie interaktywnych i dynamicznych interfejsów użytkownika.
