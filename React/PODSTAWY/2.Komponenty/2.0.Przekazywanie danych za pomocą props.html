Przekazywanie danych za pomocą props w React

W React dane są przekazywane pomiędzy komponentami za pomocą props. Propsy umożliwiają przekazywanie informacji z komponentu nadrzędnego (parent) do komponentu podrzędnego (child), co pozwala na bardziej elastyczne i dynamiczne aplikacje.

Podstawowe użycie props

Aby przekazać dane do komponentu podrzędnego, należy ustawić atrybuty w tagu komponentu w komponencie nadrzędnym, a następnie odebrać je jako argument props w komponencie podrzędnym.


        // Komponent podrzędny (Child)
        function Welcome(props) {
          return <h1>Witaj, {props.name}!</h1>;
        }

        // Komponent nadrzędny (Parent)
        function App() {
          return (
            <div>
              <Welcome name="Anna" /> {/* Przekazanie props */}
              <Welcome name="Jan" />  {/* Kolejny props */}
            </div>
          );
        }

W tym przykładzie komponent Welcome otrzymuje dane poprzez props o nazwie name, a następnie wykorzystuje te dane do dynamicznego wyświetlenia tekstu. Komponent App przekazuje różne wartości do komponentu Welcome.

Przekazywanie wielu props

Możemy przekazywać wiele propsów, które mogą mieć różne typy, takie jak liczby, łańcuchy znaków, tablice czy obiekty.


        function UserInfo(props) {
          return (
            <div>
              <p>Imię: {props.name}</p>
              <p>Wiek: {props.age}</p>
            </div>
          );
        }

        function App() {
          return (
            <div>
              <UserInfo name="Anna" age={28} />  {/* Przekazanie wielu props */}
              <UserInfo name="Jan" age={35} />
            </div>
          );
        }

W tym przykładzie UserInfo przyjmuje dwa propsy: name i age. Te dane są przekazywane z komponentu nadrzędnego App, co pozwala na wyświetlanie różnych informacji o użytkownikach.

Props jako obiekty

Zamiast przekazywać wiele pojedynczych propsów, możemy przekazać obiekt, który zawiera wszystkie potrzebne dane.


        function UserInfo({ user }) {
          return (
            <div>
              <p>Imię: {user.name}</p>
              <p>Wiek: {user.age}</p>
            </div>
          );
        }

        function App() {
          const user1 = { name: 'Anna', age: 28 };
          const user2 = { name: 'Jan', age: 35 };

          return (
            <div>
              <UserInfo user={user1} />  {/* Przekazanie obiektu jako props */}
              <UserInfo user={user2} />
            </div>
          );
        }

W tym przykładzie komponent UserInfo odbiera jeden props o nazwie user, który jest obiektem. Zawiera on wszystkie dane potrzebne do wyświetlenia informacji o użytkowniku.

Props jako funkcje

Props mogą również przekazywać funkcje, które komponent podrzędny może wywołać. To szczególnie przydatne, gdy chcemy, aby komponent podrzędny komunikował się z nadrzędnym.


        function Button(props) {
          return (
            <button onClick={props.handleClick}>
              Kliknij mnie
            </button>
          );
        }

        function App() {
          const showAlert = () => {
            alert('Przycisk został kliknięty!');
          };

          return (
            <div>
              <Button handleClick={showAlert} />  {/* Przekazanie funkcji jako props */}
            </div>
          );
        }

W tym przykładzie funkcja showAlert jest przekazywana jako props handleClick do komponentu Button. Komponent podrzędny wywołuje tę funkcję, gdy użytkownik kliknie przycisk.

Domyślne wartości props

Możemy ustawić domyślne wartości propsów w przypadku, gdy nie zostaną przekazane z komponentu nadrzędnego.


        function Welcome(props) {
          return <h1>Witaj, {props.name}!</h1>;
        }

        Welcome.defaultProps = {
          name: 'Nieznajomy'  // Ustawienie domyślnej wartości
        };

        function App() {
          return (
            <div>
              <Welcome />  {/* Props nie przekazano, użyta zostanie domyślna wartość */}
              <Welcome name="Jan" />
            </div>
          );
        }

Jeśli nie przekażemy propsa name do komponentu Welcome, zostanie użyta domyślna wartość "Nieznajomy".

PropTypes

React oferuje sposób na sprawdzanie typów propsów za pomocą biblioteki prop-types. Umożliwia to upewnienie się, że przekazujemy propsy o odpowiednich typach do komponentu.


        import PropTypes from 'prop-types';

        function UserInfo(props) {
          return (
            <div>
              <p>Imię: {props.name}</p>
              <p>Wiek: {props.age}</p>
            </div>
          );
        }

        UserInfo.propTypes = {
          name: PropTypes.string.isRequired,  // Sprawdzanie typu dla props name
          age: PropTypes.number.isRequired    // Sprawdzanie typu dla props age
        };

        function App() {
          return (
            <div>
              <UserInfo name="Anna" age={28} />
              <UserInfo name="Jan" age={35} />
            </div>
          );
        }

W tym przykładzie używamy PropTypes, aby upewnić się, że name jest ciągiem znaków, a age liczbą. Dodatkowo ustawiliśmy isRequired, co oznacza, że propsy te są wymagane.

Podsumowanie

Propsy w React to potężny sposób na przekazywanie danych między komponentami. Dzięki nim możemy tworzyć bardziej dynamiczne i wielokrotnego użytku komponenty, co pozwala na bardziej modularne podejście do tworzenia aplikacji. Możemy przekazywać wartości, obiekty, funkcje, a nawet domyślne wartości, co czyni propsy elastycznym narzędziem w React.
