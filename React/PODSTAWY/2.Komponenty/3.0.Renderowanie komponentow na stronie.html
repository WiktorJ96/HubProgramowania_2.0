Renderowanie komponentów w React

Renderowanie komponentów to proces, w którym komponenty React są zamieniane na elementy DOM, które są następnie wyświetlane w przeglądarce. React zapewnia metodę ReactDOM.render, która umożliwia renderowanie komponentów do drzewa DOM.

Podstawowe renderowanie komponentu

Najprostszym sposobem renderowania komponentu jest użycie metody ReactDOM.render. Poniżej znajduje się przykład renderowania prostego komponentu Welcome do elementu div o id root:


// Prosty komponent funkcyjny
function Welcome(props) {
  return <h1>Witaj, {props.name}!</h1>;
}

// Renderowanie komponentu
ReactDOM.render(<Welcome name="Kamil" />, document.getElementById('root'));

Renderowanie wielu komponentów

React pozwala na renderowanie wielu komponentów jednocześnie. Komponenty te mogą być zagnieżdżane w innych komponentach lub renderowane osobno. Oto przykład renderowania kilku komponentów obok siebie:


// Dwa proste komponenty funkcyjne
function Welcome(props) {
  return <h1>Witaj, {props.name}!</h1>;
}

function Goodbye(props) {
  return <p>Do zobaczenia, {props.name}!</p>;
}

// Renderowanie wielu komponentów
ReactDOM.render(
  <div>
    <Welcome name="Kamil" />
    <Goodbye name="Kamil" />
  </div>,
  document.getElementById('root')
);

Zagnieżdżanie komponentów

Komponenty React mogą być zagnieżdżane w innych komponentach, co pozwala na tworzenie bardziej złożonych struktur. Oto przykład zagnieżdżenia komponentu Greeting w komponencie App:


// Komponent Greeting
function Greeting(props) {
  return <h1>Witaj, {props.name}!</h1>;
}

// Komponent App, który zagnieżdża Greeting
function App() {
  return (
    <div>
      <Greeting name="Anna" />
    </div>
  );
}

// Renderowanie komponentu App
ReactDOM.render(<App />, document.getElementById('root'));

Renderowanie warunkowe

React pozwala na renderowanie komponentów warunkowo w zależności od wartości zmiennych lub stanu komponentu. Możemy to osiągnąć za pomocą operatora warunkowego if lub wyrażenia ternary (operator trójargumentowy). Oto przykład:


// Komponent wyświetlający powitanie lub pożegnanie na podstawie warunku
function Greeting(props) {
  const isLoggedIn = props.isLoggedIn;
  if (isLoggedIn) {
    return <h1>Witaj, użytkowniku!</h1>;
  } else {
    return <h1>Proszę się zalogować.</h1>;
  }
}

// Renderowanie warunkowego komponentu
ReactDOM.render(<Greeting isLoggedIn={true} />, document.getElementById('root'));

Renderowanie listy komponentów

React pozwala na renderowanie list komponentów dynamicznie, na przykład na podstawie tablicy danych. Ważne jest, aby każdy renderowany element listy miał unikalny atrybut key. Oto przykład renderowania listy imion:


// Komponent wyświetlający listę imion
function NameList(props) {
  const names = props.names;
  return (
    <ul>
      {names.map((name, index) => (
        <li key={index}>{name}</li>
      ))}
    </ul>
  );
}

// Lista imion do renderowania
const names = ['Anna', 'Kamil', 'Magda'];

// Renderowanie listy komponentów
ReactDOM.render(<NameList names={names} />, document.getElementById('root'));

Kiedy używać renderowania komponentów?

Renderowanie komponentów jest kluczową funkcjonalnością React. Należy go używać zawsze, gdy chcemy wyświetlić elementy interfejsu użytkownika na stronie. Możemy renderować pojedyncze komponenty, wiele komponentów, listy, a nawet renderować komponenty warunkowo w zależności od logiki aplikacji.

Podsumowanie

Renderowanie komponentów w React jest centralnym elementem tworzenia interaktywnych interfejsów użytkownika. Dzięki metodzie ReactDOM.render możemy dynamicznie tworzyć i wyświetlać komponenty w drzewie DOM. Zagnieżdżanie, warunkowe renderowanie oraz renderowanie list to ważne techniki, które umożliwiają budowanie złożonych aplikacji.
