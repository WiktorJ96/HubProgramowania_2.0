Co to jest JSX?

JSX (JavaScript XML) to rozszerzenie składni JavaScriptu, które wygląda podobnie do HTML-a i jest używane w React do opisywania, jak interfejs użytkownika powinien wyglądać. JSX ułatwia tworzenie komponentów React, umożliwiając mieszanie logiki i struktury w jednym miejscu.

Dlaczego JSX?

JSX pozwala na pisanie składni podobnej do HTML-a bezpośrednio w kodzie JavaScript, co sprawia, że komponenty React są bardziej czytelne i zrozumiałe. JSX ostatecznie jest przetwarzany przez przeglądarkę jako czysty JavaScript.

Prosty przykład JSX

Poniżej znajduje się prosty przykład JSX, który wyświetla wiadomość powitalną. JSX przypomina HTML, ale działa w środku JavaScriptu.


// Komponent funkcyjny z JSX
function Welcome(props) {
  return <h1>Witaj, {props.name}!</h1>;
}

// Użycie komponentu
ReactDOM.render(<Welcome name="Kamil" />, document.getElementById('root'));

JSX a czysty JavaScript

Pod maską JSX jest przekształcany w wywołania funkcji JavaScriptu, na przykład React.createElement. Oto jak wygląda przykład JSX w czystym JavaScripcie:


// JSX
const element = <h1>Witaj, świecie!</h1>;

// Czysty JavaScript (bez JSX)
const element = React.createElement('h1', null, 'Witaj, świecie!');

Właściwości JSX

  • Zagnieżdżanie: W JSX możesz zagnieżdżać elementy w ten sam sposób, jak w HTML.
  • Interpolacja: W JSX możesz umieszczać wyrażenia JavaScriptu wewnątrz nawiasów klamrowych {}. Jest to przydatne do dynamicznego wstawiania wartości w elementach.
  • JSX a HTML: JSX wygląda jak HTML, ale niektóre atrybuty są inne (np. className zamiast class).

Zagnieżdżanie elementów

JSX pozwala na zagnieżdżanie elementów tak jak w HTML. Oto przykład:


function App() {
  return (
    <div>
      <h1>Moje nagłówki</h1>
      <p>To jest akapit zagnieżdżony w divie.</p>
    </div>
  );
}

ReactDOM.render(<App />, document.getElementById('root'));

Interpolacja w JSX

JSX pozwala na dynamiczne wstawianie wartości JavaScriptu do elementów HTML za pomocą nawiasów klamrowych {}. Przykład:


function Welcome(props) {
  return <h1>Witaj, {props.name}!</h1>;
}

ReactDOM.render(<Welcome name="Anna" />, document.getElementById('root'));

Różnice między JSX a HTML

Mimo że JSX wygląda bardzo podobnie do HTML, istnieją pewne różnice:

  • className zamiast class: W JSX używamy className, ponieważ class jest zarezerwowanym słowem kluczowym w JavaScript.
  • onClick zamiast onclick: Eventy w JSX są pisane camelCasem (np. onClick, onChange), a nie w stylu HTML (np. onclick).
  • Fragmenty: W JSX, gdy chcemy zwrócić wiele elementów bez zagnieżdżania ich w kontenerze div, możemy użyć React.Fragment lub skróconej formy <></>.

Przykład fragmentu JSX

Fragmenty pozwalają na zwracanie wielu elementów bez dodawania dodatkowych znaczników do DOM. Przykład:


function FragmentExample() {
  return (
    <>
      <h1>To jest nagłówek</h1>
      <p>To jest akapit.</p>
    </>
  );
}

ReactDOM.render(<FragmentExample />, document.getElementById('root'));

Kiedy używać JSX?

JSX jest domyślną składnią używaną w React i jest zalecany do tworzenia komponentów interfejsu użytkownika. Jest bardziej zwięzły i czytelny niż czysty JavaScript, a także ułatwia pracę z elementami interfejsu.
