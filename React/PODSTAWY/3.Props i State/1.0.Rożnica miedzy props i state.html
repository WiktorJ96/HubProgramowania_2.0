Różnica między props a state w React

W React istnieją dwa główne mechanizmy do zarządzania danymi wewnątrz komponentów: props i state. Chociaż oba pozwalają na przechowywanie i przekazywanie informacji, działają w różny sposób i mają inne zastosowania.

Co to jest props?

props (ang. properties) to dane, które są przekazywane do komponentu z zewnątrz. Props są statyczne — komponent nie może ich zmienić, są tylko do odczytu. Są one wykorzystywane do przekazywania informacji między komponentami. Na przykład:


// Przekazywanie props
function Welcome(props) {
  return <h1>Witaj, {props.name}!</h1>;
}

// Użycie komponentu z przekazanymi props
ReactDOM.render(<Welcome name="Kamil" />, document.getElementById('root'));

W powyższym przykładzie props służy do przekazania imienia "Kamil" do komponentu Welcome, który je wyświetla.

Co to jest state?

state to dane, które są zarządzane wewnątrz komponentu. state może być zmieniany w czasie, co pozwala komponentowi na dynamiczne reagowanie na interakcje użytkownika. Zmiana stanu komponentu powoduje jego ponowne renderowanie. Przykład:


// Zarządzanie stanem komponentu
import React, { useState } from 'react';

function Counter() {
  // Inicjalizacja stanu
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Kliknięto {count} razy</p>
      <button onClick={() => setCount(count + 1)}>Kliknij mnie</button>
    </div>
  );
}

ReactDOM.render(<Counter />, document.getElementById('root'));

W tym przykładzie state jest używany do przechowywania liczby kliknięć. Za każdym razem, gdy użytkownik kliknie przycisk, stan zostaje zaktualizowany, a komponent jest renderowany ponownie, aby pokazać nową liczbę kliknięć.

Podstawowe różnice między props a state

  • Props są tylko do odczytu: Komponent nie może zmienić wartości props. Są one przekazywane do komponentu z zewnątrz i komponent korzysta z nich do wyświetlania danych.
  • State jest dynamiczny: Komponent może zmieniać swój własny state, co powoduje ponowne renderowanie komponentu, gdy stan ulega zmianie.
  • Props przekazują dane z zewnątrz: Props służą do przekazywania danych do komponentu z innego komponentu nadrzędnego.
  • State zarządza wewnętrznymi danymi: State jest wykorzystywany do zarządzania danymi wewnątrz komponentu, które mogą ulec zmianie w wyniku interakcji użytkownika lub innych czynników.

Przykład porównawczy

Oto przykład, który pokazuje zarówno użycie props, jak i state w jednym komponencie:


// Komponent funkcyjny z props i state
import React, { useState } from 'react';

function Greeting(props) {
  // Ustawienie stanu
  const [isLoggedIn, setIsLoggedIn] = useState(false);

  // Funkcja zmieniająca stan
  function handleLogin() {
    setIsLoggedIn(true);
  }

  return (
    <div>
      <h1>{isLoggedIn ? `Witaj z powrotem, {props.name}` : 'Proszę się zalogować'}</h1>
      {!isLoggedIn && <button onClick={handleLogin}>Zaloguj się</button>}
    </div>
  );
}

// Użycie komponentu z props
ReactDOM.render(<Greeting name="Kamil" />, document.getElementById('root'));

W tym przykładzie props jest używany do przekazania imienia do komponentu, a state zarządza informacją, czy użytkownik jest zalogowany. Przyciskiem zmieniamy stan, co powoduje zmianę wyświetlanego tekstu.

Kiedy używać props a kiedy state?

  • Używaj props: Gdy dane są przekazywane z komponentu nadrzędnego i komponent ma tylko wyświetlić te dane bez możliwości ich modyfikacji.
  • Używaj state: Gdy komponent musi dynamicznie zarządzać swoimi własnymi danymi i reagować na zmiany, np. w wyniku interakcji użytkownika.

Podsumowanie

Props i state to kluczowe pojęcia w React, które umożliwiają zarządzanie danymi w komponentach. Props pozwala na przekazywanie danych do komponentu z zewnątrz, natomiast state umożliwia dynamiczne zarządzanie danymi wewnątrz komponentu. Zrozumienie tych różnic pozwala na lepsze projektowanie i budowanie aplikacji React.
