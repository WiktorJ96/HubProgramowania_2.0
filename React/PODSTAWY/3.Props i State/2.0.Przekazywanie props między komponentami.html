Przekazywanie props między komponentami w React

W React, props (skrót od "properties") są sposobem na przekazywanie danych między komponentami. Props umożliwiają przesyłanie danych z komponentu nadrzędnego (rodzica) do komponentu podrzędnego (dziecka), dzięki czemu można tworzyć bardziej modularne i dynamiczne aplikacje.

Przekazywanie props z komponentu rodzica do komponentu dziecka

W React dane są przekazywane "z góry na dół", co oznacza, że komponent nadrzędny może przesłać dane do komponentu podrzędnego poprzez propsy. Oto przykład:


// Komponent rodzica
function ParentComponent() {
  const userName = "Kamil";

  return (
    <div>
      <h1>Komponent Rodzica</h1>
      {/* Przekazywanie props do komponentu dziecka */}
      <ChildComponent name={userName} />
    </div>
  );
}

// Komponent dziecka
function ChildComponent(props) {
  return <h2>Witaj, {props.name}!</h2>;
}

// Renderowanie komponentu rodzica
ReactDOM.render(<ParentComponent />, document.getElementById('root'));

W powyższym przykładzie komponent ParentComponent przekazuje userName jako props do komponentu ChildComponent. Dziecko otrzymuje dane przez props i wyświetla je w elemencie h2.

Przekazywanie wielu props

Możesz przekazywać wiele różnych propsów z komponentu rodzica do dziecka. Oto przykład:


// Komponent rodzica
function ParentComponent() {
  const userName = "Kamil";
  const userAge = 25;

  return (
    <div>
      <h1>Komponent Rodzica</h1>
      {/* Przekazywanie wielu props do komponentu dziecka */}
      <ChildComponent name={userName} age={userAge} />
    </div>
  );
}

// Komponent dziecka
function ChildComponent(props) {
  return (
    <div>
      <h2>Witaj, {props.name}!</h2>
      <p>Masz {props.age} lat.</p>
    </div>
  );
}

// Renderowanie komponentu rodzica
ReactDOM.render(<ParentComponent />, document.getElementById('root'));

W tym przykładzie komponent ParentComponent przekazuje dwa propsy: name i age. Komponent ChildComponent wyświetla oba te wartości w elemencie h2 i p.

Przekazywanie funkcji jako props

Możliwe jest także przekazywanie funkcji jako props. Dzięki temu komponent dziecka może wywołać funkcję zdefiniowaną w komponencie rodzica. Przykład:


// Komponent rodzica
function ParentComponent() {
  const handleButtonClick = () => {
    alert("Przycisk został kliknięty w komponencie dziecka!");
  };

  return (
    <div>
      <h1>Komponent Rodzica</h1>
      {/* Przekazywanie funkcji jako props */}
      <ChildComponent onButtonClick={handleButtonClick} />
    </div>
  );
}

// Komponent dziecka
function ChildComponent(props) {
  return (
    <div>
      <h2>Komponent Dziecka</h2>
      <button onClick={props.onButtonClick}>Kliknij mnie</button>
    </div>
  );
}

// Renderowanie komponentu rodzica
ReactDOM.render(<ParentComponent />, document.getElementById('root'));

W tym przypadku komponent ParentComponent przekazuje funkcję handleButtonClick jako props. Komponent ChildComponent może ją wywołać, gdy przycisk zostanie kliknięty, co spowoduje wyświetlenie alertu w przeglądarce.

Przekazywanie obiektów jako props

Możesz także przekazywać obiekty jako props, aby przekazać bardziej złożone dane. Oto przykład:


// Komponent rodzica
function ParentComponent() {
  const user = {
    name: "Kamil",
    age: 25,
    location: "Warszawa"
  };

  return (
    <div>
      <h1>Komponent Rodzica</h1>
      {/* Przekazywanie obiektu jako props */}
      <ChildComponent userInfo={user} />
    </div>
  );
}

// Komponent dziecka
function ChildComponent(props) {
  return (
    <div>
      <h2>Witaj, {props.userInfo.name}!</h2>
      <p>Masz {props.userInfo.age} lat i mieszkasz w {props.userInfo.location}.</p>
    </div>
  );
}

// Renderowanie komponentu rodzica
ReactDOM.render(<ParentComponent />, document.getElementById('root'));

W tym przykładzie obiekt user zawierający dane takie jak imię, wiek i lokalizacja jest przekazywany jako pojedynczy props userInfo. Komponent dziecka korzysta z tych danych, aby dynamicznie wyświetlić informacje.

Kiedy używać props?

Props są idealnym rozwiązaniem, gdy chcesz przekazywać dane z komponentu nadrzędnego do podrzędnego. Używaj propsów, gdy komponent dziecka musi wiedzieć coś, co zostało zdefiniowane w jego rodzicu, ale nie musi sam zarządzać tymi danymi.

Podsumowanie

Props to kluczowy mechanizm w React do przekazywania danych między komponentami. Mogą być używane do przekazywania zarówno prostych wartości, jak i bardziej złożonych obiektów czy funkcji. Zrozumienie, jak skutecznie przekazywać props między komponentami, pozwala na budowanie modularnych i elastycznych aplikacji React.
