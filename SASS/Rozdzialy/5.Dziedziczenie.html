SASS - Dziedziczenie Stylów

W SASS można dziedziczyć style między selektorami, co umożliwia użycie istniejących reguł w innych miejscach kodu bez ich powtarzania. Dzięki `@extend` stylizacje są bardziej spójne i łatwiejsze w zarządzaniu.

Podstawowe Użycie `@extend`

Aby użyć `@extend`, dodaj go w nowym selektorze, aby przejąć style z istniejącego selektora. Poniżej znajduje się przykład podstawowego dziedziczenia stylów:


// Definicja bazowego selektora
.button {
    padding: 10px 20px;
    font-size: 16px;
    color: #fff;
    background-color: #3498db;
    border: none;
    border-radius: 5px;
}

// Dziedziczenie stylów za pomocą @extend
.button-primary {
    @extend .button;
}

.button-secondary {
    @extend .button;
    background-color: #2ecc71;
}

W tym przykładzie klasa .button-primary dziedziczy wszystkie style z .button, a klasa .button-secondary dziedziczy styl z .button i dodatkowo nadpisuje kolor tła.

Dziedziczenie Stylów z Elementów Bazowych

Przy użyciu `@extend` można tworzyć elementy bazowe, które są następnie dziedziczone przez inne selektory, co pozwala tworzyć spójne struktury stylów dla całego projektu.


// Styl podstawowy dla tekstu
%base-text {
    font-family: Arial, sans-serif;
    color: #333;
    line-height: 1.5;
}

// Dziedziczenie stylu tekstu bazowego
.header {
    @extend %base-text;
    font-size: 24px;
}

.paragraph {
    @extend %base-text;
    font-size: 16px;
    margin-bottom: 10px;
}

W tym przykładzie styl %base-text jest elementem bazowym (placeholder) i nie jest sam w sobie renderowany w CSS, ale jego reguły są dziedziczone przez .header i .paragraph.

Unikanie Powielania Kodów z Placeholderami

Użycie placeholderów (elementów bazowych z symbolem %) pomaga uniknąć powielania kodów. Są renderowane tylko wtedy, gdy są dziedziczone przez inny selektor, co poprawia wydajność kodu.


// Definicja placeholdera dla przycisków
%button-style {
    padding: 10px 20px;
    font-size: 16px;
    border-radius: 5px;
    text-transform: uppercase;
}

// Dziedziczenie placeholdera
.button {
    @extend %button-style;
    background-color: #3498db;
    color: #fff;
}

.button-outline {
    @extend %button-style;
    background-color: transparent;
    color: #3498db;
    border: 2px solid #3498db;
}

W tym przykładzie placeholder %button-style jest używany przez .button i .button-outline, co pozwala na wielokrotne wykorzystanie stylów bez ich powtarzania.

Przykład Dziedziczenia w Złożonym Projekcie

Dziedziczenie stylów jest szczególnie przydatne w złożonych projektach, gdzie wiele elementów potrzebuje podobnych stylów. Poniższy przykład ilustruje zastosowanie dziedziczenia stylów dla różnych rodzajów kart:


// Styl podstawowy dla karty
%card-base {
    border: 1px solid #ddd;
    border-radius: 8px;
    padding: 15px;
    box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
}

// Dziedziczenie stylu podstawowego
.card {
    @extend %card-base;
    background-color: #fff;
}

.card-highlighted {
    @extend %card-base;
    background-color: #f9f9f9;
    border-color: #3498db;
}

W tym przykładzie .card i .card-highlighted dziedziczą styl %card-base, co zapewnia spójność i wygodę w zarządzaniu wyglądem kart.

Zalety i Ograniczenia `@extend`

**Zalety:** `@extend` pozwala na efektywne zarządzanie wspólnymi stylami i zmniejsza powtarzalność kodu. Jednak jego stosowanie wymaga ostrożności, ponieważ może generować złożone selektory, gdy używamy go wielokrotnie lub na dużą skalę.
