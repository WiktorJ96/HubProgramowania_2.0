SASS - Mixiny

Mixiny w SASS to funkcje pozwalające definiować bloki stylów, które można wielokrotnie używać w różnych miejscach projektu. Dzięki mixinom można łatwo uniknąć powtarzania kodu i sprawnie zarządzać złożonymi stylami.

Definiowanie Mixina

Aby zdefiniować mixin, użyj słowa kluczowego @mixin, a następnie nadaj nazwę mixinowi i określ bloki stylów, które chcesz wielokrotnie używać.


// Definiowanie prostego mixina
@mixin rounded-corners {
    border-radius: 10px;
    overflow: hidden;
}

// Użycie mixina
.card {
    @include rounded-corners;
    background-color: #f3f3f3;
    padding: 20px;
}

W powyższym przykładzie mixin rounded-corners zawiera styl border-radius, a jego użycie w klasie .card dodaje zaokrąglone rogi do tego elementu.

Mixiny z Parametrami

Mixiny mogą przyjmować parametry, co pozwala na dostosowywanie stylów przy każdym użyciu. Możesz na przykład przekazać wartość rozmiaru zaokrąglenia jako parametr.


// Definiowanie mixina z parametrem
@mixin rounded-corners($radius) {
    border-radius: $radius;
    overflow: hidden;
}

// Użycie mixina z różnymi wartościami
.box-small {
    @include rounded-corners(5px);
}

.box-large {
    @include rounded-corners(15px);
}

W tym przykładzie mixin rounded-corners przyjmuje parametr $radius, co pozwala na kontrolowanie promienia zaokrąglenia dla różnych elementów.

Domyślne Wartości Parametrów

Parametry w mixinach mogą mieć wartości domyślne, które zostaną użyte, jeśli nie przekażemy innej wartości podczas użycia mixina. Dzięki temu kod jest bardziej elastyczny.


// Definiowanie mixina z domyślną wartością
@mixin box-shadow($color: rgba(0, 0, 0, 0.5), $blur: 10px) {
    box-shadow: 0 0 $blur $color;
}

// Użycie mixina
.panel {
    @include box-shadow;
}

.panel-strong {
    @include box-shadow(rgba(0, 0, 0, 0.8), 15px);
}

W powyższym przykładzie mixin box-shadow ma domyślne wartości dla parametrów $color i $blur. Umożliwia to jego użycie bez dodatkowych parametrów, jak i z nimi.

Mixiny do Responsywnych Stylów

Mixiny są przydatne do definiowania stylów responsywnych. Możemy zdefiniować mixin do obsługi zapytań @media i używać go w dowolnym miejscu projektu.


// Mixin dla stylów responsywnych
@mixin respond-to($breakpoint) {
    @if $breakpoint == small {
        @media (max-width: 600px) { @content; }
    } @else if $breakpoint == medium {
        @media (max-width: 900px) { @content; }
    } @else if $breakpoint == large {
        @media (max-width: 1200px) { @content; }
    }
}

// Użycie mixina
.container {
    width: 100%;

    @include respond-to(small) {
        width: 90%;
    }

    @include respond-to(medium) {
        width: 80%;
    }
}

W przykładzie powyżej mixin respond-to dostosowuje szerokość kontenera dla różnych punktów przerwania (small, medium, large), co pozwala na łatwe zarządzanie responsywnością.

Przykład: Zaawansowany Mixin

Mixiny mogą zawierać złożone bloki stylów, które są często używane w różnych miejscach projektu. Oto przykład mixina do stylizowania przycisków z parametrami.


// Mixin do stylizacji przycisków
@mixin button-style($bg-color, $text-color: #fff, $padding: 10px) {
    background-color: $bg-color;
    color: $text-color;
    padding: $padding;
    border: none;
    border-radius: 5px;
    cursor: pointer;
    text-transform: uppercase;

    &:hover {
        background-color: darken($bg-color, 10%);
    }
}

// Użycie mixina
.button-primary {
    @include button-style(#3498db);
}

.button-secondary {
    @include button-style(#2ecc71, #333, 15px);
}

W tym przykładzie mixin button-style pozwala tworzyć spójnie stylizowane przyciski z różnymi kolorami tła, kolorem tekstu i odstępami, dzięki czemu stylizacja przycisków jest łatwiejsza do zarządzania.
