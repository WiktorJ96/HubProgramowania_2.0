Optymalizacja i Wydajność w Flutterze

Dobra wydajność aplikacji Flutter zależy od płynnego renderowania i szybkiego reagowania interfejsu. Flutter udostępnia narzędzia i techniki pozwalające wykryć i zoptymalizować problemy z wydajnością.

1. Profilowanie aplikacji

Uruchom aplikację w trybie profile aby mierzyć czas renderowania, wykorzystanie CPU i pamięci:


flutter run --profile

Do analizy danych użyj Flutter DevTools.

2. Flutter DevTools

To zestaw narzędzi do debugowania, analizy i optymalizacji wydajności.

  • Performance: analiza klatek (frame timeline)
  • Memory: monitorowanie zużycia pamięci
  • CPU Profiler: analiza operacji CPU
  • Widget Inspector: struktura i rendering UI

Uruchom DevTools z terminala:


flutter pub global run devtools

3. Minimalizacja jank (opóźnień animacji)

Jank oznacza opóźnienia w renderowaniu — np. gdy klatka trwa dłużej niż 16 ms (przy 60 FPS).

  • Unikaj ciężkich operacji w metodzie build()
  • Używaj ListView.builder zamiast ListView(children: [...])
  • Używaj lazy loading danych
  • Unikaj długich obliczeń w głównym wątku

4. RepaintBoundary

RepaintBoundary to widżet, który ogranicza ponowne renderowanie do konkretnej części UI.


RepaintBoundary(
  child: SomeHeavyWidget(),
)

Dzięki temu, jeśli coś innego się zmienia, `SomeHeavyWidget()` nie będzie się odrysowywać ponownie.

5. Dobre praktyki wydajnościowe

  • Używaj const widgetów gdzie to możliwe (const Text(...))
  • Unikaj niepotrzebnych rebuildów – rozdziel komponenty
  • Cache’uj obrazy (np. cached_network_image)
  • Używaj shouldRepaint = false w CustomPainter, jeśli nic się nie zmienia

6. Monitorowanie wydajności w produkcji

Użyj Firebase Performance Monitoring lub narzędzi do logowania np. Sentry, aby śledzić spadki FPS i błędy użytkowników.

7. Sprawdzanie wielkości bundla

Analizuj wagę aplikacji:


flutter build apk --analyze-size

Otwórz wygenerowany raport HTML w przeglądarce – zobaczysz, które pakiety zajmują najwięcej miejsca.
