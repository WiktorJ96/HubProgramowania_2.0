Podstawy Postman

Postman to popularne narzędzie do testowania API, które umożliwia wysyłanie żądań HTTP do serwera, co jest przydatne przy tworzeniu i testowaniu aplikacji działających na różnych metodach HTTP. Poniżej znajdziesz omówienie podstawowych metod HTTP oraz przykłady ich użycia w Postmanie.

Instalacja Postman

Aby rozpocząć pracę z Postmanem, można pobrać aplikację ze strony postman.com. Narzędzie to jest dostępne na systemy Windows, Mac oraz Linux i pozwala szybko testować różne API.

Wysyłanie żądań HTTP

Postman obsługuje różne metody HTTP, takie jak GET, POST, PUT i DELETE, które są używane do komunikacji z serwerem. Każda metoda ma inną funkcję:

  • GET: Pobiera dane z serwera.
  • POST: Wysyła nowe dane do serwera.
  • PUT: Aktualizuje istniejące dane na serwerze.
  • DELETE: Usuwa dane z serwera.

Przykłady użycia metod HTTP w Postman

Poniżej znajdują się przykłady użycia każdej z metod HTTP w Postmanie. Aby wysłać żądanie, należy wybrać odpowiednią metodę, wprowadzić URL API, a następnie skonfigurować wymagane nagłówki i parametry (jeśli są potrzebne).

GET


# Użycie metody GET
GET https://api.example.com/users

# Odpowiedź
{
    "users": [
        {"id": 1, "name": "Jan Kowalski"},
        {"id": 2, "name": "Anna Nowak"}
    ]
}

Metoda GET pobiera dane, takie jak lista użytkowników z serwera. Wystarczy wpisać URL i kliknąć "Send" w Postmanie.

POST


# Użycie metody POST
POST https://api.example.com/users
Body:
{
    "name": "Piotr Zieliński"
}

# Odpowiedź
{
    "id": 3,
    "name": "Piotr Zieliński"
}

Metoda POST jest używana do dodawania nowych danych. W Postmanie należy wybrać "Body" i wprowadzić dane, które chcemy przesłać.

PUT


# Użycie metody PUT
PUT https://api.example.com/users/3
Body:
{
    "name": "Piotr Z."
}

# Odpowiedź
{
    "id": 3,
    "name": "Piotr Z."
}

Metoda PUT pozwala na aktualizację istniejących danych. W przykładzie aktualizujemy dane użytkownika o ID 3.

DELETE


# Użycie metody DELETE
DELETE https://api.example.com/users/3

# Odpowiedź
{
    "message": "User deleted successfully."
}

Metoda DELETE usuwa dane, w tym przypadku użytkownika o ID 3. Wystarczy wprowadzić URL i wysłać żądanie, aby usunąć dane z serwera.

Obsługa nagłówków i parametrów

W Postmanie można także dodawać nagłówki (headers) i parametry do żądań. Są one potrzebne, aby przekazywać dodatkowe informacje, takie jak tokeny autoryzacyjne lub parametry filtrowania. Wystarczy przejść do zakładki "Headers" lub "Params" w Postmanie i dodać wymagane dane.
